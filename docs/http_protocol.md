# HTTP协议笔记
## 常见状态码
| 分类       | 状态码 | 含义        |
|----------|-----|-----------|
| 2xx成功    | 200 | ok，请求成功   |
|          | 201 | 创建成功      |
| 3xx重定向   | 301 | 永久重定向     |
|          | 302 | 临时重定向     |
|          | 304 | 未修改（使用缓存） |
| 4xx客户端错误 | 400 | 请求参数错误    |
|          | 401 | 未授权（需要登录） |
|          | 403 | 已登录（无权限）  |
|          | 404 | 资源未找到     |
| 5xx服务端错误 | 500 | 服务器内部错误   |
|          | 502 | 网关错误      |
|          | 503 | 服务不可用     |

## GET vs POST
| 特性   | GET      | POST |
|------|----------|------|
| 参数位置 | URL查询字符串 | 请求体  |
| 长度限制 | 有（浏览器限制） | 无    |
| 缓存   | 可缓存      | 不可缓存 |
| 幂等性  | 幂等       | 不幂等  |
| 安全性  | 参数暴露在URL | 相对安全 |

## Cookie 与 Session

- Cookie：存储在客户端（浏览器），大小约4kb，可被篡改，每次请求自动携带
- Session：存储在服务端，通常依赖Cookie传递Session Id，更安全但占用服务器资源

## 常见请求头

| Header        | 作用                             |
|---------------|--------------------------------|
| content-Type  | 请求/响应的媒体类型（如 application/json） |
| Authorization | 传递认证凭证（如 bearer token）         |
| User—agent    | 客户端标识（浏览器、操作系统                 |
| Cookie        | 发送存储的Cookie                    |

## 常见响应头

| Header        | 作用          |
|---------------|-------------|
| Set-Cookie    | 服务器设置Cookie |
| Location      | 重定向目标URL    |
| Cache-Control | 缓存策略        |
